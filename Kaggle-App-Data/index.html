Introduction¶

The goal of this analysis is to help an app developer better understand the app market. Specific questions asked here are: How does size of an app affect its user ratings? Does an app's price affect its popularity? How do these responses change between google and apple users?

Data sets for this analysis was obtained from Kaggle. Data for google playstore is available here and that for apple app store is available here

import numpy as np

# create array of -1.0 (size cannot be -1, so this is sort of a control)
size_bytes = -1.0 * np.ones(df_android.shape[0])
# now fill it up
megabytes = df_android['Size'][df_android['Size'].str.endswith('M')]
kilobytes = df_android['Size'][df_android['Size'].str.endswith('k')]

# verify how many were not filled
print "Number of non-quantitative sizes: ", sum(size_bytes == -1.0)
# these were the version-dependent sizes.

# now use regular expression to convert "3.0M" to "3.0"
megabytes_num = megabytes.str.findall('\d+(?:\.\d+)?')
# but, if some entries had problems, there will be more than one value.
# check that (expecting 1)
lengths = megabytes.str.findall('\d+(?:\.\d+)?').str.len()
print np.max(lengths)

# now do the conversion
for i in range(megabytes_num.shape[0]):
    size_bytes[megabytes.index[i]] = float(megabytes_num.iloc[i][0]) * 1024 * 1024
    
# now do the same for kilobytes
kilobytes_num = kilobytes.str.findall('\d+(?:\.\d+)?')
lengths = kilobytes.str.findall('\d+(?:\.\d+)?').str.len()
# expecting 1
print np.max(lengths)

for i in range(kilobytes_num.shape[0]):
    size_bytes[kilobytes.index[i]] = float(kilobytes_num.iloc[i][0]) * 1024
    
# add extra column 'size_bytes'

Next, look at the differences between paid and free apps. For this, consider the ratings and popularity. Popularity can be best judged by number of installs. This data is available for Android, not for apple. Thus, to keep things consistent, number of reviews will be used as the metric for how many people are using a given app.

A couple of observations. First, Android users seem to care a lot about whether or not an app is free. Number of reviews for paid apps is an order of magnitude lower than that for free apps. For Apple apps, this reduction is by a factor of 4. Thus, Apple users are much more open to paid apps compared to Android users.

This is also supported by the previous observation that a larger portion of the Apple apps are paid.

However, looking at the user rating, the two categories of apps fare almost similarly for both the data sets. Thus, user satisfaction is independent of app price provided the user decides to use it.
